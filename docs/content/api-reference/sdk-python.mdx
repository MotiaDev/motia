## Installation

```bash
pip install iii-sdk
```

The package is imported as `iii`:

```python
from iii import register_worker, InitOptions, ApiRequest, ApiResponse
```

## Initialization

Use `register_worker(...)` as the default bootstrap API. It creates an `III` client and starts the worker connection automatically.

```python
from iii import InitOptions, register_worker

iii = register_worker("ws://localhost:49134")

async def sum_function(data):
    a = data.get("a", 0)
    b = data.get("b", 0)
    return {"sum": a + b}

iii.register_function("math::sum", sum_function, description="Add two numbers")

result = await iii.trigger({"function_id": "math::sum", "payload": {"a": 2, "b": 3}})
print(result)  # {"sum": 5}
```

You can pass `InitOptions` to customize the worker name:

```python
from iii import InitOptions, register_worker

iii = register_worker(
    address="ws://localhost:49134",
    options=InitOptions(worker_name="my-worker"),
)
```

<ResponseField name="register_worker" type="(address: str, options: InitOptions | None = None) => III" required>
  Creates an `III` client and auto-starts the connection task using the active asyncio event loop. Use this as the default bootstrap path for workers.
</ResponseField>

<ResponseField name="III" type="class" required>
  The main low-level SDK client. Use `III(...)` directly when you need manual lifecycle control (for example explicit `await iii.connect()` timing).

  <AccordionGroup>
    <Accordion iconName="Settings" title="Constructor" description="">

      <ResponseField name="options" type="InitOptions | None">
        <Expandable title="InitOptions">

          <ResponseField name="invocation_timeout_ms" type="int">

          </ResponseField>
          <ResponseField name="enable_metrics_reporting" type="bool">

        </Expandable>
      </ResponseField>

## Methods

### connect

<ResponseField name="connect" type="async () => None" required>
  Manually establishes the WebSocket connection and re-registers all previously registered functions, triggers, and services. Use this when you initialize with `III(...)` instead of `register_worker(...)`.
</ResponseField>

### shutdown

<ResponseField name="shutdown" type="async () => None" required>
  Disconnects from the engine and cancels background tasks.
</ResponseField>

### register_function

<ResponseField name="register_function" type="(path: str, handler: Callable, description: str | None = None) => None" required>
  Registers a function with the engine.

  <AccordionGroup>
    <Accordion iconName="Settings" title="Parameters" description="">
      <ResponseField name="path" type="str" required>
        Unique function identifier.
      </ResponseField>
      <ResponseField name="handler" type="Callable[[Any], Awaitable[Any]]" required>
        Async function that processes the invocation payload.
      </ResponseField>
      <ResponseField name="description" type="str | None">
        Human-readable description.
      </ResponseField>
    </Accordion>
    <Accordion iconName="Code" title="Example" description="">
      ```python
      async def greet(data):
          return {"message": f"Hello, {data['name']}!"}

      iii.register_function("greet", greet, description="Greets a user")
      ```

    </Accordion>
  </AccordionGroup>
</ResponseField>

### register_trigger

<ResponseField name="register_trigger" type="(type: str, function_id: str, config: Any) => Trigger" required>
  Binds a trigger configuration to a registered function.

  <AccordionGroup>
    <Accordion iconName="Settings" title="Parameters" description="">

        Trigger type (e.g. `http`, `queue`, `cron`).
      </ResponseField>
      <ResponseField name="function_id" type="str" required>
        ID of the function to invoke.
      </ResponseField>
      <ResponseField name="config" type="Any" required>
        Trigger-specific configuration.
      </ResponseField>
    </Accordion>
    <Accordion iconName="ArrowUpRight" title="Returns" description="">

        Object with an `unregister()` method.
      </ResponseField>
    </Accordion>
    <Accordion iconName="Code" title="Example" description="">

    </Accordion>
  </AccordionGroup>
</ResponseField>

### trigger

<ResponseField name="trigger" type="async (request: dict) => Any | EnqueueResult | None" required>
  Invokes a remote function. The routing behavior and return value depend on the `action` key of the request dict.

  | `action` | Behavior | Return value |
  |----------|----------|--------------|
  | _(none)_ | Synchronous — waits for the function to return | function result |
  | `TriggerAction.Enqueue(...)` | Async via named queue — engine acknowledges enqueue | `EnqueueResult` |
  | `TriggerAction.Void()` | Fire-and-forget — no response | `None` |

  <AccordionGroup>
    <Accordion iconName="Settings" title="Parameters" description="">
      <ResponseField name="request" type="dict" required>
        <Expandable title="request keys">
          <ResponseField name="function_id" type="str" required>
            ID of the function to invoke.
          </ResponseField>
          <ResponseField name="payload" type="Any" required>
            Payload to pass to the function.
          </ResponseField>
          <ResponseField name="action" type="TriggerAction">
            Routing action. Omit for synchronous request/response. Use `TriggerAction.Enqueue(queue="name")` for async queue delivery or `TriggerAction.Void()` for fire-and-forget.
          </ResponseField>
          <ResponseField name="timeout_ms" type="int">
            Timeout in milliseconds for synchronous calls. Default `30000`.
          </ResponseField>
        </Expandable>
      </ResponseField>
    </Accordion>
    <Accordion iconName="Code" title="Example" description="">
      ```python
      from iii import TriggerAction

      # Synchronous — wait for function result
      result = await iii.trigger({"function_id": "get-order", "payload": {"id": "123"}})

      # Enqueue to named queue — returns EnqueueResult
      receipt = await iii.trigger({
          "function_id": "payments::charge",
          "payload": {"orderId": "123", "amount": 49.99},
          "action": TriggerAction.Enqueue(queue="payment"),
      })

      # Fire-and-forget
      await iii.trigger({
          "function_id": "notifications::send",
          "payload": {"userId": "123"},
          "action": TriggerAction.Void(),
      })
      ```
    </Accordion>
  </AccordionGroup>
</ResponseField>

### TriggerAction

A class with factory methods that construct routing actions for `trigger()`. Import from `iii`.

<ResponseField name="TriggerAction.Enqueue" type="classmethod (queue: str) => TriggerAction" required>
  Routes the invocation through a named queue. The engine enqueues the job, acknowledges the caller with an `EnqueueResult`, and processes it asynchronously. Raises an error if the queue name is not found in config or if FIFO validation fails.
</ResponseField>

<ResponseField name="TriggerAction.Void" type="classmethod () => TriggerAction" required>
  Fire-and-forget routing. The engine forwards the invocation without waiting for a response or queuing the job. No acknowledgement is returned.
</ResponseField>

### Fire-and-forget (trigger + TriggerAction.Void)

Use `trigger()` with `action: TriggerAction.Void()` for fire-and-forget invocations. No response is expected.

```python
from iii import TriggerAction

iii.trigger({
    "function_id": "notifications::send",
    "payload": {"to": "user@example.com", "subject": "Hello"},
    "action": TriggerAction.Void(),
})
```

### create_stream

<ResponseField name="create_stream" type="(stream_name: str, stream: IStream) => None" required>
  Registers a custom stream implementation. This registers `{stream_name}::get`, `::set`, `::delete`, `::list`, `::list_groups`, and `::update` functions.

  <AccordionGroup>
    <Accordion iconName="Code" title="Example" description="">
      ```python
      from iii.stream import IStream, StreamGetInput, StreamSetInput

      class MyStream(IStream):
          async def get(self, input: StreamGetInput):
              ...
          async def set(self, input: StreamSetInput):
              ...
          # ... implement delete, list, list_groups, update

      iii.create_stream("todos", MyStream())
      ```
    </Accordion>
  </AccordionGroup>
</ResponseField>

## Types

```python
from iii import ApiRequest, ApiResponse

class ApiRequest(BaseModel, Generic[TInput]):
    path_params: dict[str, str]          # alias: pathParams
    query_params: dict[str, str | list[str]]  # alias: queryParams
    body: Any | None = None
    headers: dict[str, str | list[str]]
    method: str = "GET"

class ApiResponse(BaseModel, Generic[TOutput]):
    status_code: int    # alias: statusCode
    body: Any
    headers: dict[str, str] = {}
```

<Callout title="Pydantic aliases" type="info">
  `ApiRequest` and `ApiResponse` use Pydantic `populate_by_name=True`, so both `status_code` and `statusCode` work when constructing instances.
</Callout>

## Context

```python
from iii import Logger

logger = Logger()
logger.info("Processing started")
```

The `Logger()` function returns the current `Context` (with `logger`) from within a function handler. The logger automatically sends log events to the engine.
