title: 'KV Server'
description: 'Key-Value storage server for persistent data storage'
---

Key-Value storage server for persistent data storage with support for in-memory and file-based backends.

```
modules::kv_server::KvServer
```

## Sample Configuration

```yaml
- class: modules::kv_server::KvServer
  config:
    store_method: file_based
    file_path: ./data/kv_store
    save_interval_ms: 5000
```

## Configuration

<ResponseField name="store_method" type="string">
  Storage backend. `in_memory` (default) — data is lost on restart. `file_based` — data is persisted to disk and survives restarts.
</ResponseField>

<ResponseField name="file_path" type="string">
  Directory path for `file_based` storage. Each index is written as a separate file. Defaults to `kv_store_data.db`.
</ResponseField>

<ResponseField name="save_interval_ms" type="number">
  Interval in milliseconds between disk writes when using `file_based` storage. Defaults to `5000`.
</ResponseField>

<ResponseField name="kv_server::update" type="function">
  Update a value by key in the KV store using patch operations. Creates the key with an empty object if it does not exist.

  <AccordionGroup>
    <Accordion iconName="settings" title="Parameters">
      <ResponseField name="index" type="string" required>
        The index (namespace) to use for organizing data.
      </ResponseField>
      <ResponseField name="key" type="string" required>
        The key of the value to update.
      </ResponseField>
      <ResponseField name="ops" type="UpdateOp[]" required>
        An array of patch operations to apply to the value. Each operation has a `type` and a `path`:

        | Type | Fields | Description |
        |------|--------|-------------|
        | `set` | `path`, `value` | Set a field at the given path. Empty path replaces the whole value. |
        | `merge` | `value` | Shallow-merge an object into the root value. |
        | `increment` | `path`, `by` | Increment a numeric field by `by`. Creates the field if missing. |
        | `decrement` | `path`, `by` | Decrement a numeric field by `by`. Creates the field if missing. |
        | `remove` | `path` | Remove a field at the given path. |
      </ResponseField>
    </Accordion>
    <Accordion title="Returns">
      <ResponseField name="old_value" type="any">
        The value before the update, or `null` if the key did not exist.
      </ResponseField>
      <ResponseField name="new_value" type="any">
        The value after applying all operations.
      </ResponseField>
    </Accordion>
  </AccordionGroup>
</ResponseField>

## Usage Examples

### Basic Key-Value Operations

```typescript
await iii.trigger('kv_server::set', {
  index: 'default',
  key: 'user:123',
  value: { name: 'John', email: 'john@example.com' },
})

const user = await iii.trigger('kv_server::get', {
  index: 'default',
  key: 'user:123',
})

await iii.trigger('kv_server::delete', {
  index: 'default',
  key: 'user:123',
})
```

### Listing Indexes with Prefix

`list_keys_with_prefix` returns **index names** (namespaces), not keys within an index. Use it when your index names follow a naming convention and you want to discover which indexes exist.

```typescript
await iii.trigger('kv_server::set', { index: 'user:1', key: 'profile', value: { name: 'Alice' } })
await iii.trigger('kv_server::set', { index: 'user:2', key: 'profile', value: { name: 'Bob' } })
await iii.trigger('kv_server::set', { index: 'product:1', key: 'info', value: { name: 'Widget' } })

const userIndexes = await iii.trigger('kv_server::list_keys_with_prefix', {
  prefix: 'user:',
})
// returns ['user:1', 'user:2']
```
