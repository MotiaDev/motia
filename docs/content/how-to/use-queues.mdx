title: 'Use Queues'
description: 'How to enqueue work and process it asynchronously with retries, concurrency control, and ordering guarantees.'

## Goal

Offload work to a queue so it can be processed asynchronously by worker functions — with built-in retries, concurrency control, and optional ordering.

## Steps

### 1. Enable the Queue module

```yaml title="iii-config.yaml"
modules:
  - class: modules::queue::QueueModule
    config:
      adapter:
        class: modules::queue::BuiltinQueueAdapter
        config:
          store_method: file_based      # Options: in_memory, file_based
          file_path: ./data/queue_store # Required for file_based
          max_attempts: 3               # Max retries before DLQ (default: 3)
          backoff_ms: 1000              # Base backoff delay in ms (default: 1000)
          concurrency: 10               # Concurrent workers (default: 10)
          poll_interval_ms: 100         # Poll interval in ms (default: 100)
          mode: "concurrent"            # "fifo" or "concurrent" (default: "concurrent")
```

These are global defaults. Individual workers can override `maxRetries`, `backoffDelayMs`, `concurrency`, and `type` per subscription (see step 4).

### 2. Enqueue work

From any function, push a job onto a queue by calling `enqueue` with a `topic` (the queue name) and a `data` payload. The call is fire-and-forget — the caller returns immediately while the queue stores the job for processing.

<Tabs groupId="language" persist items={["Node / TypeScript", "Python", "Rust"]}>
<Tab value="Node / TypeScript">
```typescript title="enqueue-order.ts"
import { init, Logger } from 'iii-sdk'

const iii = init(process.env.III_BRIDGE_URL ?? 'ws://localhost:49134')

iii.registerFunction({ id: 'orders::create' }, async (input) => {
  const logger = new Logger()
  const order = { id: crypto.randomUUID(), ...input }

  iii.triggerVoid('enqueue', {
    topic: 'order.created',
    data: order,
  })

  logger.info('Order enqueued', { orderId: order.id })
  return { orderId: order.id }
})
```
</Tab>
<Tab value="Python">
```python title="enqueue_order.py"
import uuid
import os
from iii import III, Logger

iii = III(os.environ.get('III_BRIDGE_URL', 'ws://localhost:49134'))

async def create_order(input):
    logger = Logger()
    order = {"id": str(uuid.uuid4()), **input}

    iii.trigger_void("enqueue", {
        "topic": "order.created",
        "data": order,
    })

    logger.info('Order enqueued', {'orderId': order['id']})
    return {'orderId': order['id']}

iii.register_function("orders::create", create_order)
```
</Tab>
<Tab value="Rust">
```rust title="enqueue_order.rs"
use iii_sdk::{Logger, III};
use serde_json::json;

let iii = III::new(&std::env::var("III_BRIDGE_URL").unwrap_or_else(|_| "ws://127.0.0.1:49134".to_string()));

let iii_clone = iii.clone();
iii.register_function("orders::create", move |input| {
    let iii = iii_clone.clone();
    async move {
        let logger = Logger();
        let order_id = uuid::Uuid::new_v4().to_string();

        iii.trigger_void("enqueue", json!({
            "topic": "order.created",
            "data": { "id": order_id, "items": input["items"], "total": input["total"] },
        }))?;

        logger.info("Order enqueued", Some(json!({ "orderId": order_id })));
        Ok(json!({ "orderId": order_id }))
    }
});

```
</Tab>
</Tabs>

### 3. Register a worker

Bind a function to a queue using a `queue` trigger. The engine invokes the function once per job. When the worker returns successfully the job is acknowledged and removed. If the worker throws, the job is retried according to the queue configuration.

<Tabs groupId="language" persist items={["Node / TypeScript", "Python", "Rust"]}>
<Tab value="Node / TypeScript">
```typescript title="process-order.ts"
import { init, Logger } from 'iii-sdk'

const iii = init(process.env.III_BRIDGE_URL ?? 'ws://localhost:49134')

iii.registerFunction({ id: 'orders::process-payment' }, async (job) => {
  const logger = new Logger()
  logger.info('Processing payment', { orderId: job.id })
  // ...payment logic...
  return { processed: true }
})

iii.registerTrigger({
  type: 'queue',
  function_id: 'orders::process-payment',
  config: { topic: 'order.created' },
})
```
</Tab>
<Tab value="Python">
```python title="process_order.py"
import os
from iii import III, Logger

iii = III(os.environ.get('III_BRIDGE_URL', 'ws://localhost:49134'))

async def process_payment(job):
    logger = Logger()
    logger.info('Processing payment', {'orderId': job['id']})
    # ...payment logic...
    return {'processed': True}

iii.register_function("orders::process-payment", process_payment)
iii.register_trigger(
    type="queue",
    function_id="orders::process-payment",
    config={"topic": "order.created"},
)
```
</Tab>
<Tab value="Rust">
```rust title="process_order.rs"
use iii_sdk::{Logger, III};
use serde_json::json;

let iii = III::new(&std::env::var("III_BRIDGE_URL").unwrap_or_else(|_| "ws://127.0.0.1:49134".to_string()));

iii.register_function("orders::process-payment", |job| async move {
    let logger = Logger();
    let order_id = job["id"].as_str().unwrap_or("");
    logger.info("Processing payment", Some(json!({ "orderId": order_id })));
    // ...payment logic...
    Ok(json!({ "processed": true }))
});

iii.register_trigger("queue", "orders::process-payment", json!({
    "topic": "order.created",
}))?;
```
</Tab>
</Tabs>

A worker can also enqueue further work, creating processing pipelines:

```typescript
iii.registerFunction({ id: 'orders::process-payment' }, async (job) => {
  // ...charge the customer...

  iii.triggerVoid('enqueue', {
    topic: 'order.paid',
    data: { orderId: job.id },
  })

  return { processed: true }
})
```

### 4. Configure queue behavior

Override global defaults per worker by passing `queue_config` in the trigger:

```typescript
iii.registerTrigger({
  type: 'queue',
  function_id: 'orders::process-payment',
  config: {
    topic: 'order.created',
    queue_config: {
      maxRetries: 5,
      backoffType: 'exponential',
      backoffDelayMs: 2000,
      concurrency: 5,
      type: 'fifo',
    },
  },
})
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `maxRetries` | `number` | `3` | Max delivery attempts before moving to DLQ |
| `backoffType` | `string` | `"exponential"` | Backoff strategy |
| `backoffDelayMs` | `number` | `1000` | Base delay between retries (ms) |
| `concurrency` | `number` | `10` | Max concurrent jobs for this worker |
| `type` | `string` | `"concurrent"` | `"concurrent"` processes jobs in parallel; `"fifo"` processes them in order |

### 5. Filter jobs before processing

Use `condition_function_id` to skip jobs that don't meet a condition. The condition function receives the job payload and returns a boolean — only truthy results proceed to the worker:

<Tabs groupId="language" persist items={["Node / TypeScript", "Python", "Rust"]}>
<Tab value="Node / TypeScript">
```typescript title="filtered-worker.ts"
const isHighValue = iii.registerFunction({ id: 'orders::is-high-value' }, async (job) => {
  return job.total > 1000
})

iii.registerFunction({ id: 'orders::high-value-alert' }, async (job) => {
  const logger = new Logger()
  logger.info('High-value order', { orderId: job.id, total: job.total })
  return { alerted: true }
})

iii.registerTrigger({
  type: 'queue',
  function_id: 'orders::high-value-alert',
  config: {
    topic: 'order.created',
    condition_function_id: isHighValue.id,
  },
})
```
</Tab>
<Tab value="Python">
```python title="filtered_worker.py"
async def is_high_value(job):
    return job.get("total", 0) > 1000

async def high_value_alert(job):
    logger = Logger()
    logger.info('High-value order', {'orderId': job['id'], 'total': job['total']})
    return {'alerted': True}

iii.register_function("orders::is-high-value", is_high_value)
iii.register_function("orders::high-value-alert", high_value_alert)

iii.register_trigger(
    type="queue",
    function_id="orders::high-value-alert",
    config={
        "topic": "order.created",
        "condition_function_id": "orders::is-high-value",
    },
)
```
</Tab>
<Tab value="Rust">
```rust title="filtered_worker.rs"
let high_value_check = iii.register_function("orders::is-high-value", |job| async move {
    let total = job["total"].as_f64().unwrap_or(0.0);
    Ok(json!(total > 1000.0))
});

iii.register_function("orders::high-value-alert", |job| async move {
    let logger = Logger();
    let id = job["id"].as_str().unwrap_or("");
    let total = job["total"].as_f64().unwrap_or(0.0);
    logger.info("High-value order", Some(json!({ "orderId": id, "total": total })));
    Ok(json!({ "alerted": true }))
});

iii.register_trigger("queue", "orders::high-value-alert", json!({
    "topic": "order.created",
    "condition_function_id": high_value_check.id,
}))?;
```
</Tab>
</Tabs>

### 6. Trigger the workflow

With the queue, worker, and any configuration in place, kick off the workflow by calling the function that creates work:

<Tabs groupId="language" persist items={["Node / TypeScript", "Python", "Rust"]}>
<Tab value="Node / TypeScript">
```typescript
const { orderId } = await iii.trigger('orders::create', { items: ['item-1', 'item-2'], total: 99.99 })
logger.info('Order enqueued', { orderId })
```
</Tab>
<Tab value="Python">
```python
result = await iii.trigger('orders::create', {'items': ['item-1', 'item-2'], 'total': 99.99})
print('Order enqueued', result['orderId'])
```
</Tab>
<Tab value="Rust">
```rust
let result = iii.trigger("orders::create", json!({ "items": ["item-1", "item-2"], "total": 99.99 })).await?;
println!("Order enqueued: {}", result["orderId"]);
```
</Tab>
</Tabs>

## Result

Jobs are enqueued instantly and processed asynchronously by worker functions. Failed jobs retry with exponential backoff. After all retries exhaust, jobs move to a dead letter queue where they are preserved for inspection or reprocessing.

  See [Manage Failed Triggers](./dead-letter-queues) for DLQ configuration and redrive.
</Callout>
